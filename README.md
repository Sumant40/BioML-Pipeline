## Overview

BioML Pipeline takes real tumor biopsy data (RNA-Seq gene expression measurements) and classifies which type of cancer each patient has. It chains five Docker containers in sequence — each doing one job — and produces an ensemble prediction that combines a TensorFlow neural network and a PyTorch MLP.

**Achieved accuracy: 100% on 161 real TCGA patient test samples across 5 cancer types.**

---

## Biology Background

Every cell in your body expresses genes at different levels depending on what the cell is doing. Cancer cells have abnormal gene expression patterns that differ by cancer type — breast tumors activate different genes than kidney tumors, for example.

**RNA-Seq (RNA Sequencing)** measures the activity level of every gene in a tissue sample simultaneously. The result is a vector of ~20,000 numbers per patient — one per gene — called a **gene expression profile**.

This project uses those profiles as features to train classifiers that can identify which cancer type a patient has purely from their gene expression data.

| Cancer code | Full name | Organ |
|---|---|---|
| BRCA | Breast invasive carcinoma | Breast |
| KIRC | Kidney renal clear cell carcinoma | Kidney |
| COAD | Colon adenocarcinoma | Colon |
| LUAD | Lung adenocarcinoma | Lung |
| PRAD | Prostate adenocarcinoma | Prostate |

## Dataset

**UCI PANCAN RNA-Seq Gene Expression Dataset**

- **Source:** UCI Machine Learning Repository
- **URL:** https://archive.ics.uci.edu/dataset/401/gene+expression+cancer+rna+seq
- **Patients:** 801 tumor samples
- **Features:** 20,531 gene expression values per patient
- **Classes:** 5 cancer types (BRCA, KIRC, COAD, LUAD, PRAD)
- **Missing values:** None
- **Original data:** The Cancer Genome Atlas (TCGA)

### Download steps

1. Go to https://archive.ics.uci.edu/dataset/401/gene+expression+cancer+rna+seq
2. Click **Download**
3. Extract `TCGA-PANCAN-HiSeq-801x20531.tar.gz`
4. Place both `data.csv` and `labels.csv` into the `data/` folder

## Architecture

```
data/data.csv + data/labels.csv
          │
          ▼
┌─────────────────────┐
│  Container 1        │  python:3.11-slim
│  datagen            │  Merges CSVs, encodes labels 0-4
│                     │  → data/gene_expression.csv
└────────┬────────────┘
         │ depends_on: datagen
         ▼
┌─────────────────────┐
│  Container 2        │  python:3.11-slim + openjdk-21 + pyspark
│  spark              │  Spark: validates file, checks nulls
│                     │  pandas: StandardScaler, 80/20 split
│                     │  → data/processed/train.csv + test.csv
└────────┬────────────┘
         │ depends_on: spark
    ┌────┴────┐
    ▼         ▼
┌───────┐ ┌─────────┐   both run in parallel
│  tf   │ │pytorch  │
│       │ │         │
│Dense  │ │  MLP    │
│Neural │ │ Custom  │
│  Net  │ │  Loop   │
└───┬───┘ └────┬────┘
    │           │  depends_on: tensorflow + pytorch
    └─────┬─────┘
          ▼
┌─────────────────────┐
│  Container 5        │  python:3.11-slim
│  ensemble           │  Averages TF + PyTorch probabilities
│                     │  → final cancer type prediction
└─────────────────────┘
```

Data flows through **bind-mounted host folders** (`./data`, `./models`) so all output files are visible directly on your machine after the pipeline runs.

## Setup & Run

### 1. Clone / create the project

```bash
# If using git
git clone <your-repo-url>
cd bioml-pipeline

# Or just create the folder and add all files manually
mkdir bioml-pipeline && cd bioml-pipeline
```

### 2. Download the dataset

Place `data.csv` and `labels.csv` from the UCI link above into the `data/` folder.

```
bioml-pipeline/
└── data/
    ├── data.csv      ← must be here before running
    └── labels.csv    ← must be here before running
```

### 3. Create the models folder

Docker cannot create host folders on Windows automatically.

```bash
mkdir models
```

### 4. Run the pipeline

```bash
docker compose up
```

That's it. Docker pulls all images on first run (~3 GB, takes 5-10 min on good internet), then runs the full pipeline. Subsequent runs skip the download.

### 5. View results

Results print to the terminal from the `ensemble` container. Trained models are saved to `models/` and processed data to `data/processed/`.

## Troubleshooting

### `bitnami/spark:3.5: not found`
Bitnami's Spark image was paywalled by Broadcom in September 2025. This project uses `python:3.11-slim` + `openjdk-21-jdk-headless` + `pyspark` pip instead. Make sure you're using the latest `docker-compose.yml`.

### `FileNotFoundError: data/processed/train.csv`
The `spark` container didn't write its output where `tensorflow`/`pytorch` can read it. Check that all three services use `./data:/app/data` (bind mount), not `bioml_data` (named volume). Run `docker compose down -v` to wipe old volumes, then `docker compose up`.

### `Py4JNetworkError: Answer from Java side is empty`
The JVM ran out of memory during Spark ML operations. The current `preprocess.py` avoids all Spark ML (no VectorAssembler, no Spark StandardScaler) — make sure you're using the latest version which uses sklearn for normalisation instead.

### Docker Desktop 502 Bad Gateway
Docker Desktop's WSL2 engine crashed from memory pressure. Restart Docker Desktop, increase memory in Settings → Resources → Memory (6 GB+ recommended), then `docker compose down && docker compose up`.

### `maxColumns` error / `ArrayIndexOutOfBoundsException`
Spark's default CSV column limit is 20,480. The PANCAN dataset has 20,532 columns. The `spark.read.option("maxColumns", "25000")` call in `preprocess.py` fixes this — make sure it's present.

### `version` attribute warning
`docker-compose.yml` has no `version:` field — this is intentional. Modern Docker Compose treats it as obsolete and ignores it, but older versions warn. Safe to ignore.

## Docker Reference

```bash
# Run full pipeline
docker compose up

# Run in background (detached)
docker compose up -d

# Watch logs live
docker compose logs -f

# Watch one service only
docker compose logs -f spark

# Stop everything
docker compose down

# Stop and delete volumes (use when volume bugs suspected)
docker compose down -v

# Rebuild from scratch
docker compose down -v && docker compose up --build

# Run one service manually
docker compose run --rm tensorflow

# Open a shell inside a container (for debugging)
docker compose run --rm spark bash

# List downloaded images
docker images

# Free disk space (removes all unused images)
docker system prune -a
```
